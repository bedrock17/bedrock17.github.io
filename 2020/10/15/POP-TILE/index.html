POPTILE

친구가 만들었던게임이 개인적으로 재미있어 그 게임을 웹으로 다시 만드는 토이프로젝트를 진행하였습니다.

~ 서비스 링크 http://s0af.panty.run/ ~
바뀐 서비스 링크 https://poptile.firekann.dev/

당시에는 친구가 무엇을 보고 만들었었는지, 기존에 이런 프로그램이 있었는지 몰랐었지만 찾아보니 바로 무엇을 보고 만들었는지 알 수 있었습니다.

오히려 처음부터 원본게임을 안보고 만들었기에 개인적인 의견, 플레이해본 다른사람들의 의견을 가지고 재밌게 개발한거같네요. 메인로직만 비슷하지 배치, 테마, 점수산정, 효과 모든게 다 다르게 되었습니다.

초기 구현

jsfiddle: https://jsfiddle.net/_cube/qpo4jvx3/

2017년도에 이 코드를 작성했던거 같은데 HTML Canvas태그와 자바스크립트를 사용해 간단한 게임을 만드는것에 관심이 있어 만들만한걸 생각하다가 친구의 게임을 생각해 만들었었습니다.

현재의 제가 당시 코드를 평가하자면.. 요즘 자바스크립트 코드와 스타일이 약간 다르고, 실제로 그랬지만 일단 뭔가 실행되는 걸 보고싶어 구현만 신경썻다 라는 느낌을 받습니다. 모든 변수 선언 키워드가 var이고 구현에 필요한 변수들도 모두 전역변수에 놓은점이 유독 눈에 띄네요.

그래도 상수 역할을 하는 변수와, 각 코드의 역할이 명확하여 나쁘지 않은거같습니다.

다시 만들게 된 앞배경

시간이 흘러 개인 프로젝트를 진행할 때 주력 무기(?) 가 될만한 도구를 찾고있었습니다.
아래 요구사항을 가능한 만족하는 도구들을 찾고있었습니다.

  • 빠른 프로젝트 구성
    C, C++, 기타 무언가 설치할게 많은 프레임워크 및 언어 들은 프로젝트 초기 구성이 너무 번거롭다 느껴졌습니다. 그렇지 않은 언어와 프레임워크를 찾고있었죠.
    후보 언어로는 Go, 파이썬, Nodejs 등이 있었습니다.

  • 어디서든 개발가능하고 실행가능한 프로젝트
    라즈베리파이, 윈도우 PC, 우분투 서버 등, 아키텍처, OS 모두 다른 환경을 가지고있었기에 가능한 어디서든 돌리기도 쉽고 개발환경이 없는곳에서조차 빌드된 결과물만 가지고 실행하고 싶었습니다.
    이쯤에서 Go는 확정적이였습니다.

  • 개발하기도 쉽고, 접근성 좋은 UI
    개발은 편하게 했다치고 이제 만든 프로그램을 누군가와 공유해야 하니 빌드된 바이너리를 배포하던 앱을 스토어에 올린다던가 웹서비스를 연다던가 하는 배포 문제와 쉬운 UI개발을 할 방법을 찾고있었습니다.
    브라우저만 있다면 볼 수 있는 웹이 가장 적합했고 당시 제 기준에서 인기 많고 쉬운 Vue를 사용하기로 하였습니다.

진행 및 결과

이제 정한 도구들로 무언가를 만들어야 하는데 이왕이면 사람들의 관심을 끌 수 있고 이미 제가 만들어놓은걸 활용하고 싶어 묵혀두었던 jsfiddle에 있던 팝타일 코드와 “Go 언어 웹 프로그래밍 철저 입문” 이라는 프레임워크 없이 내장 http모듈을 가지고 웹서비스를 만드는 책을 참고하여 만들다가 만 조잡한 자체 웹 프레임워크로 개발을 시작하였습니다.

팝타일 코드를 타입스크립트로 바꾸고 다듬은 다음 Vue로 작성된 UI에 연결하였습니다.
서버가 하는일이 굉장히 단순하여 조잡한 도구로도 특별히 지장이 없었습니다.
API 요청이 오면 데이터를 처리하고 Vue로 빌드된 정적 파일을 날라주고 하는일이 그것 외 없었기에 무난히 구현되어 주변 지인및 오픈채팅방에 링크를 배포했고 수십명가량 플레이 해주었습니다.

초기 배포는 집에 설치되어있던 라즈베리파이를 가지고 서비스를 하였는데 이를 안타깝게 여긴 지인이 팀블로그인 panty.run에 도커로 제가 제어 가능한 부분을 할당해주었고 그곳에 서비스가 올라가 지금 구성이 완성되었습니다.
